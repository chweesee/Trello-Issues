[Archived] [Huawei specific] Sending a message with a non-image file attached using the native Huawei file explorer will break the attachment (https://trello.com/c/rowWZfZA)

P3 - Will be fixed eventually

Created: 2017-11-13T23:43:11.133Z
Latest Update: 2019-06-10T20:43:18.600Z

Reported by Alex1304#9704

####Steps to reproduce:
- Go to a channel you have perms to send messages with attachments 
- Click the `+` button 
- Click the Import icon at top left 
- In the popup, select your native Huawei file explorer 
- Find a file that is NOT an image and select it (e.g PDF or ZIP files). Also works with *some* images located in a folder containing a . nomedia file. To reproduce this step you can also long press on any file or folder in your explorer and then tap "Compress" to quickly create a ZIP file 
- Send the message with the attachment

####Expected result:
The attachment is sent normally in chat
####Actual result:
the filename is replaced by a number and the file extension is gone
####Client settings:
Discord Alpha 5.5.1
####System settings:
Huawei Mate 9 Android 7.0
